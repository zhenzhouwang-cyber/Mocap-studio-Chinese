# 动捕查看器功能提升方案

> 目标：让使用更便捷、可视化更强，并符合常规 3D 软件（Blender / Maya / Vicon Nexus）操作习惯。  
> 与项目规划「下一步开发方向分析」衔接：在「参考比对 + 双序列对比」之前，先稳定单序列 3D 查看体验。

---

## 一、现状简要

- **已实现**：Three.js 3D 视窗、OrbitControls（拖拽旋转/滚轮缩放）、固定 bounds、地面网格、骨骼线+关节球、播放/暂停、时间显示、帧滑块、速度调节、下载 HTML / 浏览器中打开。
- **不足**：缺少快捷键与视图预设、无轨迹尾迹与关节标签、时间轴不可点击跳转、无全屏/独立窗口优化、无参考动作叠放，与「常规 3D 软件习惯」仍有差距。

---

## 二、设计原则

1. **符合常规 3D 软件习惯**：视图预设（前/侧/顶/透视）、快捷键（空格播放、F 聚焦、数字键视图）、中键/右键平移、可选世界坐标轴与网格。
2. **更便捷**：时间轴点击跳转、帧号输入、循环/区间播放、界面布局与控件位置符合「时间轴在下、视图在上」的常见布局。
3. **更可视化**：轨迹尾迹、关节/骨骼标签（悬停或可选显示）、可选多选 marker 高亮、为后续「参考动作叠放」预留数据与渲染接口。

---

## 三、提升项与优先级

### P0：交互与操作习惯（必做）✅ 已实现

| 项目 | 说明 | 实现要点 |
|------|------|----------|
| **视图预设** | 前视图(1)、侧视图(2)、顶视图(3)、透视(4)，一键切换 | ✅ 数字键 1–4 切换；F 聚焦、R 重置 |
| **快捷键** | 空格=播放/暂停，F=聚焦到骨骼包围盒，R=重置视角 | ✅ keydown 监听，输入框聚焦时不拦截 |
| **时间轴点击跳转** | 点击时间轴任意位置跳转到该时刻 | ✅ 时间轴条可点击，换算为帧索引并跳转 |
| **帧号输入** | 输入框输入帧号或时间(秒)后跳转 | ✅ 帧号输入框，回车/失焦后跳转 |
| **循环播放** | 勾选后播放到最后一帧再从头播放（当前已是循环，可改为「单次/循环」切换） | ✅ 「循环」勾选：循环 / 单次播完停止 |

### P1：可视化增强

| 项目 | 说明 | 实现要点 |
|------|------|----------|
| **蒙皮骨骼** | 用圆柱体绘制每段骨骼，替代细线，动画更直观 | ✅ 已实现：`viewer_3d` 支持 `skin_bones`，GUI 显示选项可勾选「蒙皮骨骼」 |
| **轨迹尾迹** | 可选显示最近 N 帧的轨迹线（如 30 帧），便于看运动走向 | ✅ 显示选项可选 0/15/30/60/90/120 帧尾迹，每关节一条半透明线 |
| **关节/骨骼标签** | 悬停关节时显示 marker 名称；可选「始终显示」模式 | ✅ 勾选「悬停显示关节名」后，鼠标悬停关节显示 tooltip |
| **世界坐标轴** | 可选显示 XYZ 轴（如原点处短箭头），便于对齐与教学 | ✅ 勾选「世界坐标轴」显示 AxesHelper 于场景中心 |
| **网格与地面** | 保持现有地面网格，增加「显示/隐藏」开关 | ✅ 勾选「地面网格」控制 GridHelper.visible |

### P2：布局与窗口体验 ✅ 已实现

| 项目 | 说明 | 实现要点 |
|------|------|----------|
| **标准布局** | 视图区占满上方，时间轴+播放控制在下方一条栏，与 Blender/DCC 一致 | ✅ body flex 列布局，`#viewport` flex:1，控件栏固定在下 |
| **全屏** | 按钮或快捷键(如 F11)进入全屏视窗 | ✅ 「全屏」按钮 + F11，Fullscreen API，退出时恢复 |
| **独立窗口** | 当前已有「在浏览器中打开」，可补充「新窗口打开并传递 JSON」避免大 HTML 复制 | 保留当前「写临时 HTML 再打开」方式 |

### P3：为参考比对与双序列预留

| 项目 | 说明 | 实现要点 |
|------|------|----------|
| **双序列数据结构** | 支持传入「当前 + 参考」两套 frames/labels/segments，用不同颜色渲染 | `mocap_to_viewer_json` 支持 `reference_data` 或第二组 frames；HTML 内两套骨骼（如主绿色、参考半透明蓝） |
| **时间对齐显示** | 参考比对时可按 DTW 对齐帧对应显示（当前帧 vs 参考帧） | 传入 `frameMapping: [curIdx -> refIdx]`，播放时当前用 curIdx，参考用 refIdx 更新 |
| **透明度与开关** | 参考骨骼/轨迹可调透明度，可单独关闭当前或参考 | UI：两个 checkbox + 参考透明度滑块 |

以上 P3 可与路线 A「参考动作 + DTW 评分」同步设计接口，实现可排在 A1 之后。

## 四、技术实现要点（与现有代码衔接）

- **入口**：仍以 `viewer_3d.build_3d_viewer_html()` 为唯一入口，通过新增可选参数传入「尾迹长度、是否显示坐标轴、是否显示标签、参考数据」等，保持向后兼容。
- **数据**：`mocap_to_viewer_json()` 已输出 `frames/labels/segments/bounds/frameRate/nFrames`，无需改结构即可做轨迹尾迹（前端按当前帧回溯取点）；视图预设仅用 `bounds` 算相机位置。
- **前端**：在现有 Three.js 单文件中扩展：
  - 视图预设：预计算 4 组 `camera.position` + `controls.target`，按键切换。
  - 轨迹尾迹：为每个 joint 维护一个 `Points` 或 `Line` 对象，每帧 push 当前位置并保留最近 N 个点。
  - 标签：用 `THREE.Raycaster` + `raycaster.intersectObjects(jointMeshes)` 检测悬停，用 DOM overlay 或 CSS2DRenderer 显示名称。
- **快捷键**：在 `document` 或 `canvas` 上 `keydown`，避免在 input/range 获得焦点时冲突（可选：仅在 viewport 获得焦点时响应）。

---

## 五、建议实施顺序

1. **Phase 1（P0）**：视图预设 + 快捷键（空格/F/R）+ 时间轴点击跳转 + 帧号输入 + 循环/单次切换。  
   → 先达到「常规 3D 软件习惯」与「操作便捷」。
2. **Phase 2（P1）**：轨迹尾迹 + 关节悬停标签 + 世界坐标轴 + 网格显示开关。  
   → 增强可读性与教学/分析用途。
3. **Phase 3（P2）**：布局调整 + 全屏 + 独立窗口体验（按需）。  
   → 提升长时间使用的舒适度。
4. **Phase 4（P3）**：与参考比对功能联动，双序列数据结构与可选时间对齐显示。  
   → 与计划中的「参考动作 + DTW」路线 A 衔接。

## 六、验收标准（简要）

- 用户无需看说明即可：用空格播放/暂停、用数字键 1–4 切换视图、用 F 聚焦到动作、用时间轴点击或帧号跳转。
- 可选打开轨迹尾迹与关节标签后，能快速辨认关键 marker 与运动方向。
- 后续接入「参考动作」时，可在同一视窗中并排或叠放当前与参考，且可调透明度与开关。

---

## 七、视窗比例与全屏（已落实）

- **16:9 横版视窗**：3D 内容区域固定为 16:9 比例，在视图区内居中显示；两侧或上下留黑边以保持比例，不拉伸。
- **全屏自适应**：全屏时以 `#viewport` 为全屏元素，内部 16:9 的 `#viewbox` 根据窗口尺寸重新计算宽高并居中；`fullscreenchange` 时用 `requestAnimationFrame` 延迟一帧再更新相机与 renderer 尺寸，避免布局未完成导致偏左上或尺寸错误。

## 八、Mixamo 蒙皮方案 ✅ 已实现

当前除**几何蒙皮**（圆柱体骨骼）外，已支持 **Mixamo 通用 3D 人形**蒙皮驱动。

| 步骤 | 内容 |
|------|------|
| **模型来源** | Mixamo 网站导出 GLB，骨骼命名需与动捕线段一致（如 `Mixamorig:Hips`、`Mixamorig:LeftArm`）。 |
| **加载** | 查看器内用 Three.js `GLTFLoader` 加载 GLB，自动查找 `SkinnedMesh` 与 `Skeleton`。 |
| **数据衔接** | `mixamo_retarget.build_mixamo_bone_mapping(segments, labels)` 从线段构建「骨骼名 → start/end marker」；前端每帧根据当前帧 marker 位置算 world 位置与朝向，转本地空间后写 `bone.position`/`bone.quaternion`，再 `skeleton.updateMatrixWorld(true)`。 |
| **使用** | 在动捕查看器「⚙️ 显示选项」中填写 **Mixamo 蒙皮模型 URL (GLB)**，填入可访问的 GLB 地址即可；不填则仍显示骨骼/圆柱蒙皮。 |

实现位置：`opera_mocap_tool/mixamo_retarget.py`、`viewer_3d.build_3d_viewer_html(mixamo_glb_url=...)` 与查看器内 GLB 加载与每帧驱动逻辑。
